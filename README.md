# ColorPicker Studio - 桌面色彩取样与品牌规范工具

一款专为设计师、品牌负责人和前端开发打造的桌面色彩工具。支持从任意窗口吸取颜色，整理品牌色板，检查色彩规范。

## 原始需求

> 扩展屏幕放大取样：用户按快捷键进入取色模式后，鼠标周围显示放大镜、像素网格、当前颜色值和最近取样，点击后颜色进入当前色板，按 Esc 能无痕退出并回到原窗口。
> 扩展对比度校验：选择文字色和背景色后，工具显示对比度、可读性等级、适用字号和不推荐场景，若不通过，给出同色系微调建议并保留原品牌色。

## 功能特性

### 屏幕取色（放大镜取样）
- 🎨 **全局快捷键唤起** - Alt + Shift + C 快速进入取色模式
- 🔍 **放大镜预览** - 鼠标周围显示像素级放大预览，精准定位
- 📐 **像素网格** - 放大镜内叠加像素网格，中心像素高亮出框
- 🎯 **实时颜色值** - 跟随鼠标显示 HEX、RGB、HSL、颜色名称、坐标
- 🕐 **最近取样** - 取色过程中显示本次已取的颜色，便于对比参考
- ⌨️ **无痕退出** - 按 Esc 或右键立即退出，不留下任何痕迹，回到原窗口
- 🖱️ **点击取色** - 左键点击确认取色，自动添加到当前色板和历史记录

### 对比度校验
- 📊 **对比度比值** - 实时计算 WCAG 标准对比度比值
- 🏆 **可读性等级** - 优秀 / 良好 / 一般 / 较差 四档评估
- 👥 **适用人群** - 基于对比度推荐适合的用户群体
- ⏱️ **阅读时长** - 评估适合的阅读场景时长
- 🔤 **适用字号** - 普通文本、大标题、粗体文本、图标图形分别判断
- ⚠️ **不推荐场景** - 根据对比度列出不适用的具体使用场景
- 💡 **智能配色建议** - 对比度不足时，提供同色系微调方案
- 🎨 **保留品牌色** - 建议保持色相不变，仅微调明度/饱和度
- ✅ **一键应用建议** - 点击即可应用推荐的优化配色
- 🎯 **目标等级选择** - AA 大文本 / AA 普通 / AAA 三档可选

### 其他功能
- 🎯 **多格式显示** - HEX、RGB、HSL 多种格式一键切换复制
- 📋 **剪贴板集成** - 点击即复制，高效流畅
- 🎨 **色板管理** - 创建多个色板，整理品牌色彩系统
- ⭐ **收藏夹** - 收藏常用颜色，快速访问
- 📜 **历史记录** - 自动记录取色历史，最多保存100条
- 📤 **多格式导出** - 支持 CSS 变量、SCSS、JSON、Tailwind 配置导出
- 💾 **本地存储** - 所有数据本地持久化保存

## 启动方式

### 前置要求

- Node.js >= 16.0.0
- npm 或 yarn 或 pnpm
- Windows / macOS / Linux 桌面环境

### 启动步骤

#### 1. 安装依赖

```bash
npm install
```

#### 2. 启动开发模式

```bash
npm run electron:dev
```

开发模式会同时启动 Vite 开发服务器和 Electron 窗口，支持热更新。

#### 3. 生产构建

```bash
npm run electron:build
```

构建后会在 `dist` 目录生成安装包。

### 快捷键

- `Alt + Shift + C` - 全局取色（进入放大镜取色模式）
- `Escape` - 取消取色 / 无痕退出取色模式
- 鼠标左键 - 确认取色（颜色自动加入当前色板）
- 鼠标右键 - 取消取色
- 点击颜色卡片 - 复制 HEX 值

## Docker 启动方式

> **注意**：Electron 是桌面 GUI 应用，Docker 中运行需要 X11 显示服务器支持。Docker 方式主要用于开发环境统一和构建，不建议用于日常使用。

### 前置要求

- Docker >= 20.10
- Docker Compose >= 2.0
- Linux 环境下需要 X11 显示服务器
- Windows/macOS 下需要额外配置显示转发（如 X410、XQuartz）

### Docker 启动步骤

#### 1. 构建并启动开发容器

```bash
docker compose up --build
```

后台运行：

```bash
docker compose up --build -d
```

#### 2. 访问开发服务器

开发服务器地址：http://localhost:5173

> 注意：Docker 容器内无法直接启动 Electron 桌面窗口，只能访问 Web 版前端界面。完整的桌面应用体验（包括屏幕取色、全局快捷键等）请在本地环境运行。

#### 3. 停止服务

```bash
docker compose down
```

### Docker 构建生产版本

```bash
docker compose run --rm app npm run electron:build
```

## 使用说明

### 放大镜取色操作

1. 按下 `Alt + Shift + C` 或点击界面上的"开始取色"按钮
2. 进入取色模式后：
   - 鼠标**上方**显示 160×160 像素放大镜，内含像素网格和中心高亮像素
   - 鼠标**右下方**显示信息面板，包含：颜色预览、颜色名称、HEX、RGB、HSL、坐标
   - 信息面板底部显示本次取色会话中已取的最近颜色（最多 8 个）
   - 信息面板自动避开屏幕边缘（超出边界时自动翻转到另一侧）
3. 移动鼠标到目标颜色上，通过放大镜和像素网格精确定位
4. **点击鼠标左键**确认取色，颜色自动添加到当前色板和历史记录
5. 按 **Esc** 键或**鼠标右键**无痕退出取色模式，回到原窗口

### 对比度检查

1. 切换到"对比度"页面
2. 输入或选择前景色（文字色）和背景色
3. 实时查看：
   - 对比度比值和 WCAG 等级（AAA / AA / Fail）
   - 可读性评估（优秀/良好/一般/较差）及适用人群、阅读时长
   - 四种类型的适用字号判断（普通文本、大标题、粗体、图标）
   - 不推荐场景警示（根据对比度级别动态列出）
   - 实时预览区域（大字号、标题、正文、小号、按钮）
4. 若对比度不达标：
   - 页面底部会显示"智能配色建议"区域
   - 提供 1-3 种同色系微调方案（保持色相不变，仅调整明度/饱和度）
   - 每个方案显示原颜色与建议色对比、调整后的对比度、调整方式说明
   - 标注"品牌色保留"说明（色相完全保留等）
   - 点击"应用此建议"按钮即可一键替换对应颜色
5. 可切换目标等级（AA 大文本 / AA 普通 / AAA），建议会相应调整

### 色板管理

1. 在"色板"页面点击"+ 新建"创建新色板
2. 点击色板标签切换不同色板
3. 双击颜色名称可以重命名
4. 取色时颜色自动添加到当前激活的色板

### 导出色板

1. 切换到"导出"页面
2. 选择要导出的色板
3. 选择导出格式（CSS/SCSS/JSON/Tailwind）
4. 点击"复制代码"或"下载文件"

## 注意事项

- 首次启动可能需要几秒钟加载
- 取色功能需要屏幕录制权限（macOS）
- 所有数据保存在本地，不上传云端
- Docker 方式仅用于开发和构建，完整功能（屏幕取色、全局快捷键）需本地运行 Electron 应用
